# 실행 파일 vs 설치 프로그램 차이점

## 1. 송장번호일괄처리시스템.exe (단일 실행 파일)

### 특징
- **설치 불필요**: 더블클릭으로 바로 실행
- **포터블**: USB에 넣어서 어디서든 실행 가능
- **간단함**: 설치 과정 없음

### 생성 방법
```bash
pyinstaller 송장번호일괄처리시스템.spec
# 또는
pyinstaller build_installer.spec
```

### 위치
- `dist/송장번호일괄처리시스템.exe`

### 장점
- 빠른 배포
- 설치 권한 불필요
- 간단한 테스트에 적합

### 단점
- 시작 메뉴에 등록 안 됨
- 제거 프로그램 목록에 없음
- 설정 파일 위치가 불명확할 수 있음

---

## 2. 송장번호 일괄 처리 시스템_Setup_v1.0.0.exe (설치 프로그램)

### 특징
- **전문적인 설치**: Windows 표준 설치 마법사
- **시작 메뉴 등록**: 프로그램 목록에 표시
- **제거 프로그램**: Windows에서 깔끔하게 제거 가능
- **바로가기 자동 생성**: 바탕화면, 시작 메뉴

### 생성 방법
1. 먼저 실행 파일 빌드:
   ```bash
   python build_installer.py
   ```

2. Inno Setup으로 설치 프로그램 생성:
   ```bash
   "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss
   ```

### 위치
- `installer/송장번호 일괄 처리 시스템_Setup_v1.0.0.exe`

### 장점
- 전문적인 느낌
- 사용자 친화적
- 표준 Windows 프로그램처럼 동작
- 제거가 깔끔함

### 단점
- 빌드 과정이 2단계
- Inno Setup 설치 필요

---

## 어떤 것을 사용할까?

### 단일 실행 파일 (.exe) 사용 시기
- 빠른 테스트
- 포터블 버전 필요
- 설치 권한 없는 환경
- 간단한 배포

### 설치 프로그램 (.exe) 사용 시기
- 정식 배포
- 고객에게 제공
- 전문적인 느낌 필요
- 표준 Windows 프로그램처럼 동작해야 할 때

---

## 권장 사항

**일반 사용자에게는 설치 프로그램을 권장합니다:**
- 더 전문적이고 신뢰감 있음
- Windows 표준 방식
- 제거가 쉬움
- 시작 메뉴에서 쉽게 찾을 수 있음
