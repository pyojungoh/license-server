# 문제 해결 가이드 - "불은 들어오는데 컴퓨터가 인식 안 됨"

## 🔍 현재 상황
- ✅ ESP32에 불이 들어옴 (전원 공급 정상)
- ❌ 컴퓨터에서 인식 안 됨 (드라이버 문제)

## 🚨 즉시 확인 사항

### Step 1: 장치 관리자 확인 (가장 중요!)

1. **`Win + X` 키 누르기**
2. **장치 관리자** 선택
3. 다음을 확인:

#### ✅ 정상인 경우:
- **포트(COM & LPT)** → **USB-SERIAL CH340 (COM3)** 같은 항목이 보임
- → **이 경우 드라이버는 설치되어 있습니다!** 다른 문제일 수 있습니다.

#### ❌ 문제인 경우 (가장 흔함):
- **다른 장치** 또는 **알 수 없는 장치** 섹션에 **느낌표(!)** 표시
- 또는 **USB 컨트롤러** 섹션에 **알 수 없는 USB 장치**
- → **드라이버가 설치되지 않았거나 잘못 설치된 상태**

## 🔧 해결 방법

### 방법 1: 드라이버 설치 (가장 먼저 시도)

1. **ESP32를 PC에서 분리** (USB 케이블 뽑기)

2. **드라이버 다운로드**:
   - 링크: http://www.wch.cn/downloads/CH341SER_EXE.html
   - **CH341SER.EXE** 파일 다운로드

3. **드라이버 설치**:
   - 다운로드한 **CH341SER.EXE** 실행
   - 관리자 권한 요청 시 **예** 클릭
   - **INSTALL** 버튼 클릭
   - "The driver has been installed successfully!" 메시지 확인

4. **ESP32 다시 연결**

5. **장치 관리자 새로고침** (F5 키)

6. **포트(COM & LPT)** 확인:
   - ✅ **USB-SERIAL CH340 (COMx)** 보이면 성공!

### 방법 2: 기존 드라이버 제거 후 재설치

만약 방법 1이 안 되면:

1. **ESP32를 PC에서 분리**

2. **장치 관리자** 열기

3. **보기 → 숨겨진 장치 표시** 체크

4. **다른 장치** 또는 **알 수 없는 장치** 섹션에서:
   - **알 수 없는 장치** 또는 **느낌표(!)** 항목 찾기
   - **우클릭 → 장치 제거**
   - **"이 장치의 드라이버 소프트웨어 삭제"** 체크 (있는 경우)
   - **제거** 클릭

5. **PC 재부팅** (선택사항이지만 권장)

6. **드라이버 재설치** (방법 1의 2-3단계)

7. **ESP32 다시 연결**

### 방법 3: USB 케이블/포트 문제 확인

드라이버가 정상인데도 안 되면:

1. **다른 USB 케이블 시도**
   - 데이터 전송이 가능한 케이블인지 확인 (충전 전용 케이블은 안 됨)

2. **다른 USB 포트 시도**
   - USB 2.0 포트 사용 권장
   - USB 3.0 포트도 시도해보기

3. **USB 허브 사용 중이면 직접 연결**
   - USB 허브를 거치지 말고 PC에 직접 연결

### 방법 4: Windows 업데이트 확인

1. **설정** → **Windows 업데이트**
2. **업데이트 확인** 클릭
3. 업데이트가 있으면 설치 후 재부팅

## 🔍 장치 관리자에서 확인하는 방법 (상세)

### 정상 상태:
```
포트(COM & LPT)
  └─ USB-SERIAL CH340 (COM3)  ← 이렇게 보이면 정상!
```

### 문제 상태:
```
다른 장치
  └─ 알 수 없는 장치 (!)  ← 이렇게 보이면 드라이버 문제

또는

USB 컨트롤러
  └─ 알 수 없는 USB 장치 (!)  ← 이것도 드라이버 문제
```

- [ ] 드라이버 설치 시도

- [ ] ESP32 다시 연결

- [ ] 다른 USB 케이블 시도
- [ ] 다른 USB 포트 시도

## 🆘 여전히 안 되면

1. **PC 재부팅** 후 다시 시도
2. **다른 PC에서 테스트** (ESP32가 정상인지 확인)
3. **ESP32 제품 문제 가능성** (다른 ESP32로 테스트)
